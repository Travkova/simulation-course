### Пуассоновский поток. События на сервере
Моделирование потока заявок на сервер.

**Задание:**
- событие — поступление заявки;
- определить число заявок за интервал времени T;
- построить распределение числа заявок;
- вычислить среднее и дисперсию;
- сделать вывод.

### Отчёт по выполненной работе
*Ключевые алгоритмы*

1. Простейший поток событий обладает тремя ключевыми свойствами:

- **Независимость** – наступление событий не зависимо друг от друга.
- **Ординарность** – в один момент времени может наступить не более одного события.
- **Стационарность** – вероятность числа событий в интервале зависит только от длины интервала, а не от его начальной точки.

2. *Распределение числа событий*: Для простейшего потока число событий $X$ за время $t$ распределено по закону Пуассона: $$P(X=m) = \frac{(\lambda t)^m e^{-\lambda t}}{m!}, \quad m = 0, 1, 2, \dots$$, где $\lambda$ – интенсивность потока.
3.  *Интервалы между событиями*: Интервалы $\tau_i$ между последовательными событиями имеют экспоненциальное распределение с параметром $\lambda$. Генерация интервалов выполняется по формуле обратного преобразования: $$\tau = -\frac{\ln \alpha}{\lambda}, \quad \alpha \sim U(0,1)$$
4. *Теоретические числовые характеристики*: Числовые характеристики числа событий за время $T$: $$M[X] = \lambda T, \quad D[X] = \lambda T$$
5. *Метод статистической обработки*: Для статистической обработки в работе использован **Алгоритм 2** из лекции:
   - Моделируется длинная реализация потока за время $T_1 \gg T$.
   - Случайно выбираются $N$ начальных точек $a \in [0, T_1 - T]$.
   - Для каждой точки подсчитывается число событий в интервале $[a, a+T]$.

*Вывод:* В ходе лабораторной работы было выполнено моделирование потока запросов на сервер с использованием модели простейшего (пуассоновского) потока событий. 
Были заданы следующие параметры:
* Интенсивность потока ($\lambda$): **5.0**
* Интервал времени ($T$): **10.0**
* Число экспериментов ($N$): **2000**
* Общее время моделирования ($T1$): **5000**

На графике распределения числа запросов на сервер видно, что гистограмма эмпирических частот визуально совпадает с теоретическим. Теоретическое матожидание и дисперсия составляют $\lambda \times T = 50.0$. Эмпирическое среднее (**50.0580**) и дисперсия (**50.5806**). Погрешность не превышает 1%, что объясняется конечным объёмом выборки и подтверждает закон больших чисел.
