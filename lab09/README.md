### Система массового обслуживания M/M/1

Моделирование простейшей системы M/M/1.  

### Отчёт по проделанной работе

**Система М/М/1 характеризуется:**
| Параметр | Описание |
| -------- | -------- |
| Входящий поток| Простейший (Пуассоновский) с интенсивностью $\lambda$. Интервалы между поступлениями заявок имеют экспоненциальное распределение|
| Время обслуживания | Имеет экспоненциальное распределение с интенсивностью $\mu$. Среднее время `1 / μ` |
| Кол-во каналов| 1 + 0 мест в очереди → система с потерями|
| Дисциплина | При занятом сервере новая заявка получает отказ и покидает систему |

**Ключевые формулы:**
| Показатель | Формула |
|------------|---------|
| Загрузка системы | `ρ = λ / μ` |
| Вероятность того, что сервер свободен | `P₀ = 1 / (1 - ρ)` |
| Вероятность того, что сервер занят | `P₁ = ρ / (1 + ρ)` |
| Абсолютная пропускная способность системы (количество успешно обработанных заявок в единицу времени) | `A = λ · P₀` |

**Выводы**: При параметрах $\lambda = 5$, $\mu = 6$  , $N = 1000$ (кол-во моделируемых заявок) получаются следующие результаты:
- Приведенная нагрузка $\rho = 5/6 \approx 0.833$.
- Теоретические значения: $P_0 \approx 0.5455$, $P_1 \approx 0.4545$.
При разном количестве заявок получаются следующие результаты:

| Количество заявок $N$ | Эмп. $P_0$ | Эмп. $P_1$ | Эмп. $A=λ×P_0$ | $\Delta P_0$ | $\Delta P_1$ | $\Delta A$ |
| ---------------------- | --------- | ---------- | -------------- | ------------ | ------------ | ---------- |
| 100 | 0.4900 | 0.5100 | 2.4500 | 0.0555 | 0.0555 | 0.2773 |
| 1000 | 0.5180 | 0.4820 | 2.5900 | 0.0275 | 0.0275 | 0.1373 |
| 10000 | 0.5363 | 0.4637 | 2.6815 | 0.0092 | 0.0092 | 0.0458 |
| 100000 | 0.5444 | 0.4556 | 2.7220 | 0.0011 | 0.0011 | 0.0053 |

Можно сделать вывод, что при увеличении объёма выборки N в 10 раз абсолютная погрешность уменьшается примерно в $\sqrt{10} = 3.16$ раз, что согласуется с ЦПТ. Также можно заметить, что эмпирические характеристики сходятся к теоретическим при N>10000, что подтверждает корректность алгоритма моделирования.
